hk manages your git hooks and project lints with an emphasis on **performance**. Unlike other hook managers, hk runs linters in parallel using file-level read/write locks — so you get maximum concurrency without race conditions.

## How it works

<Steps>
  <Step title="Install hk">
    Install via mise, Homebrew, Cargo, or Aqua — hk is a single Rust binary with no runtime dependencies.
    ```bash
    mise use hk
    ```
  </Step>
  <Step title="Initialize your project">
    Run `hk init` to generate a `hk.pkl` configuration file in your repository root.
    ```bash
    hk init
    ```
  </Step>
  <Step title="Configure your linters">
    Edit `hk.pkl` to define hooks and steps using built-in linters or your own commands.
    ```pkl
    amends "package://github.com/jdx/hk/releases/download/v1.41.0/hk@1.41.0#/Config.pkl"
    import "package://github.com/jdx/hk/releases/download/v1.41.0/hk@1.41.0#/Builtins.pkl"

    hooks {
      ["pre-commit"] {
        fix = true
        stash = "git"
        steps {
          ["prettier"] = Builtins.prettier
          ["eslint"] = Builtins.eslint
        }
      }
    }
    ```
  </Step>
  <Step title="Install and run">
    Install the git hooks, then commit as normal — hk runs your linters in parallel automatically.
    ```bash
    hk install
    git commit -m "feat: add feature"
    ```
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Parallel execution" icon="arrow-right-arrow-left">
    Runs all linters concurrently with file-level read/write locks — safely, without race conditions.
  </Card>
  <Card title="120+ built-in linters" icon="database">
    Prettier, ESLint, Ruff, Cargo Clippy, Golangci-lint, and 100+ more — ready to use instantly.
  </Card>
  <Card title="Smart stashing" icon="floppy-disk">
    Three-way merge stashing protects your unstaged work-in-progress during hook runs.
  </Card>
  <Card title="Type-safe Pkl config" icon="shield-check">
    Apple's Pkl language gives you schema validation, IDE autocomplete, and composable configs.
  </Card>
  <Card title="mise integration" icon="wrench">
    Deep integration with mise-en-place for tool management, task running, and env vars.
  </Card>
  <Card title="Zero runtime deps" icon="cube">
    Single Rust binary. Built-in evaluator (pklr) means no pkl CLI required.
  </Card>
</CardGroup>
